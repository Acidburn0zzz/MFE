Installation

Before starting with the installation of the MFE, you must be aware that some of the tools in the MFE require an extended version of Maude (with some additional hooks providing required functionalities) and external tools that are not available in all platforms.

The current available tools are:

  • Church-Rosser Checker (CRC)
  • Coherence Checker (ChC)
  • Maude Termination Tool (MTT)
  • Sufficient Completeness Checker (SCC)
  • Inductive Theorem Prover (ITP)

Common installation for Linux/OS X

ITP, CRC, ChC and MTT are available for both Linux and OS X. However, SCC relies on the CETA library, which is only available on Linux. The installation follows the same procedure for both platforms.

Required files

The MFE can be used both on the official distribution of Maude or on the Maude++ extended version (with appropriate hooks). However, the MTT and the SCC require internal hooks not available in the official distribution, and therefore these tools will not be available inside the MFE unless the extended version of Maude is used. The rest of the tools will work normally on both versions of Maude. The tools will give an error message if you try to use them when not available.

The following files, available from the Downloads section, are required to install the MFE:

  • MFE (Maude executable specifications of the environment and all available tools)
  • Extended version of Maude (only necessary if the MTT and SCC tools are to be used)

The MTT requires the installation of external back-end termination tools that follow the rules of the termination competition (the AProVE tool, in its 1.2 pre-release version is available here).

Installation

Uncompress the required files to the directories of choice.

Termination Back-End Tools

Automatic termination checks by the MTT will require back-end tools, like CiME, AProVE, MuTerm, etc. If there are not back-end tools configured properly, then a advisory message will be displayed by the MFE whenever a check of termination is attempted. Any tool that adheres to the specification for participants into the annual termination competition should work.

The interaction with the different termination tools is carried out following ideas already in the MTT.

Configuration of Back-End Tools

The information of the available back-end tools is read from the file mfe.config. The MFE searches for this file in maude++'s executable directory, and then in the directory referenced by the shell variable MAUDE_LIB. Such a configuration file must contain one text line per tool, and with the following syntax: 

      < name-with-which-you-will-refer-to-the-tool-inside-MFE > < path-of-the-batch-of-the-tool > < extension-of-the-filesâ€“to-load >

For instance, the mfe.config file could look like this: 

      aprove /usr/local/share/runme .trs
      ttc /usr/local/share/runmepar

The third parameter in each text line is optional. See the mfe.config file included in maude++.tar.gz for more details.
